# Contributing to the Bitcoin Sovereignty Research Framework

Thank you for your interest in contributing to this repository. This project maintains a high standard of legal, philosophical, and operational clarity. All contributions should support the framework’s core mission of advancing Bitcoin sovereignty through rigorous, professional research.

---

## Core Principles

All work in this repository must remain consistent with the **Bitcoin Sovereignty Core Principles v1.0**. Contributions that contradict these principles will not be accepted.

---

## Document Standards

All documents submitted to this repository must comply with the **Bitcoin Sovereignty Publishing Baseline v1.0**. This includes:

- Clear structure and section headings
- Professional, precise language
- Proper versioning
- Alignment with the two-tier model (Foundations and Operational)
- Accurate legal and factual referencing

Please review the Publishing Baseline before submitting new documents or significant updates.

---

## How to Contribute

1. **Fork the repository** and create a new branch for your changes.
2. **Make your changes** following the document standards above.
3. **Ensure consistency** with existing canonical documents, particularly the Core Principles and Global Framework Document.
4. **Submit a Pull Request** with a clear description of the changes and their purpose.

---

## Types of Contributions Welcome

- Refinements or expansions to existing Tier 1 or Tier 2 documents
- New nation-specific analysis (following the established structure)
- Updates to the Case Law Compendium
- Improvements to policy templates or supporting materials
- Corrections to factual or legal inaccuracies

---

## What We Do Not Accept

- Content that contradicts the Core Principles
- Speculative or low-fidelity analysis
- Marketing, promotional, or hype-driven language
- Documents that do not meet the standards set in the Publishing Baseline

---

## Versioning

Major documents use semantic versioning. Significant changes to content or structure should result in a version increment. Minor formatting or consistency updates may retain the existing version number with an updated date.

---

## Questions

If you are unsure whether a contribution aligns with the framework’s direction, please open an issue first to discuss it.
